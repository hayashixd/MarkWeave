## 1. 概要と目的

### 1.1 概要

ワークスペース（フォルダ）が Git リポジトリである場合に、エディタ内で Git の状態（変更・未追跡・コンフリクト）を可視化し、簡易的なコミットと差分確認を行える機能。

### 1.2 目的・設計思想

- **Git の状態把握**: 外部ターミナルを開かなくても、どのファイルに変更があるかをファイルツリー上で即座に確認できる
- **最小限の操作性**: 本格的な Git GUI クライアントを置き換えるものではなく、執筆中に最低限必要な「状態確認」「差分確認」「コミット」に絞る
- **エンジニア・テクニカルライター向け**: Markdown を Git 管理している技術者層に特化した機能

## 2. 機能スコープ

### 2.1 提供する機能

| 機能 | 詳細 |
|------|------|
| Git 状態バッジ（ファイルツリー） | M（変更）・U（未追跡）・A（追加済み）・D（削除）・C（コンフリクト）を表示 |
| 差分（Diff）表示 | ガター（行番号横）の差分インジケーターとインライン Diff ビュー |
| 簡易コミット | ステージング + コミットメッセージ入力 + コミット実行 |
| コミット履歴表示 | 直近のコミット一覧と各コミットの変更ファイル |
| ブランチ表示 | 現在チェックアウトしているブランチ名のステータスバー表示 |

### 2.2 提供しない機能（スコープ外）

- `git push` / `git pull`（認証情報の管理が複雑なため、将来検討）
- マージ・リベース操作
- コンフリクト解決ツール（差分表示のみ）
- ブランチ切り替え・作成

## 3. ファイルツリーの Git 状態表示

### 3.1 バッジ表示

ファイルツリーの各ファイル・フォルダにインデックスの状態バッジを表示する。

```
ファイルツリー:

📁 docs/
│  📄 system-design.md        [M]  ← 変更あり（修正）
│  📄 roadmap.md                   ← 変更なし（表示なし）
│  📄 new-feature.md          [U]  ← 未追跡（新規ファイル）
│  📄 deleted-file.md         [D]  ← 削除済み
📁 src/
│  📄 app.tsx                 [A]  ← ステージング済み
│  📄 conflict.ts             [C]  ← コンフリクトあり
📄 README.md                  [M]
```

### 3.2 バッジの色分け

| バッジ | 意味 | 色 |
|-------|------|-----|
| `M` | 変更（Modified）— ワーキングツリー | オレンジ |
| `U` | 未追跡（Untracked） | 緑 |
| `A` | ステージング済み（Added） | 緑 |
| `D` | 削除（Deleted） | 赤 |
| `C` | コンフリクト（Conflict） | 赤・太字 |
| `R` | リネーム（Renamed） | オレンジ |

### 3.3 フォルダのバッジ

フォルダには配下の変更ファイル数の集計を表示する。

```
📁 docs/  [3]   ← 配下に変更ファイル 3 件
```

## 4. タブバー・ステータスバーの状態表示

### 4.1 タブバーの状態

タブタイトルの横に Git 状態バッジを表示する。

```
[📄 system-design.md  M ✕]   ← M = 変更あり
[📄 app.tsx           A ✕]   ← A = ステージング済み
```

### 4.2 ステータスバーの Git 情報

ステータスバー右端にブランチ名と変更件数を表示する。

```
┌─────────────────────────────────────────────────────────────────┐
│ Markdown  行 42, 列 10  UTF-8  1,234 文字  ⎇ main  M:3 U:2     │
└─────────────────────────────────────────────────────────────────┘
                                          ↑ブランチ ↑M(変更)3件 U(未追跡)2件
```

- `⎇ main` をクリック → Git パネル（サイドバー）を開く
- `M:3 U:2` をクリック → Git パネルを開く

## 5. 差分（Diff）表示

### 5.1 ガターの差分インジケーター

エディタの行番号の横（ガター）に差分インジケーターを表示する。

```
  1 │ ## 設計概要
  2 │
+ 3 │ 新しく追加した行                  ← 緑の追加マーカー
  4 │ 変更されていない行
~ 5 │ 変更された行の内容                ← 青のオレンジマーカー
  6 │ 変更されていない行
- 7 │                                    ← 赤の削除マーカー（削除された行の位置に表示）
  8 │ 変更されていない行
```

| マーカー | 意味 | 色 |
|---------|------|-----|
| `+` | 追加された行 | 緑背景 |
| `~` | 変更された行 | オレンジ背景 |
| `-` | 削除された行の境界 | 赤の小バー |

### 5.2 インライン Diff ビュー

ガターのマーカーをクリックすると、その行の変更前後を表示するポップアップを表示する。

```
クリック後のポップアップ:

┌─────────────────────────────────────────────────────────────┐
│ - 変更前のテキストがここに表示されます（赤・打ち消し線）    │
│ + 変更後のテキストがここに表示されます（緑・下線）          │
├─────────────────────────────────────────────────────────────┤
│ [元に戻す]  [ステージング]  [閉じる]                        │
└─────────────────────────────────────────────────────────────┘
```

## 6. 簡易コミット機能

### 6.1 Git パネル（サイドバー）

サイドバーの「Git」タブに変更ファイル一覧とコミット UI を表示する。

```
┌──────────────────────────────────────────────────────┐
│ Git  ⎇ main                              [更新 🔄]  │
├──────────────────────────────────────────────────────┤
│ コミットメッセージ                                   │
│ ┌──────────────────────────────────────────────────┐ │
│ │ feat: スラッシュコマンド設計を追加               │ │
│ └──────────────────────────────────────────────────┘ │
│  [コミット] [ステージング + コミット]               │
├──────────────────────────────────────────────────────┤
│ ステージング済み (1)                    [全解除]    │
│   ✓ 📄 slash-commands-design.md      [A] [差分]    │
├──────────────────────────────────────────────────────┤
│ 変更 (2)                         [すべてステージ]   │
│   📄 system-design.md             [M] [+] [差分]   │
│   📄 roadmap.md                   [M] [+] [差分]   │
├──────────────────────────────────────────────────────┤
│ 未追跡 (1)                       [すべてステージ]   │
│   📄 new-design.md                [U] [+] [差分]   │
└──────────────────────────────────────────────────────┘
```

### 6.2 ステージング操作

| 操作 | 動作 |
|------|------|
| `[+]` ボタン | そのファイルをステージングに追加（`git add`）|
| `[-]` ボタン | ステージングから解除（`git restore --staged`）|
| `[すべてステージ]` | 変更/未追跡ファイルを全てステージング（`git add -A`）|
| `[全解除]` | ステージングを全て解除 |

### 6.3 コミット実行

```typescript
// コミットフロー
async function commit(message: string, stageAll: boolean) {
  if (stageAll) {
    await invoke('git_stage_all');  // git add -A
  }
  if (message.trim() === '') {
    showError('コミットメッセージを入力してください');
    return;
  }
  await invoke('git_commit', { message });
  // インデックス更新・ステータスバー更新
}
```

## 7. Git ログ表示

### 7.1 コミット履歴パネル

Git パネルの下部またはタブ切り替えで「履歴」ビューを表示する。

```
┌──────────────────────────────────────────────────────────────┐
│ 履歴  ⎇ main                                                │
├──────────────────────────────────────────────────────────────┤
│ ● feat: スラッシュコマンド設計を追加    2026-02-25  [abc123]│
│   AI テンプレートとの統合設計を含む                          │
│                                                              │
│ ● fix: Diff 表示のバグを修正           2026-02-24  [def456]│
│                                                              │
│ ● docs: ロードマップを更新              2026-02-23  [ghi789]│
└──────────────────────────────────────────────────────────────┘
```

- コミットをクリック: そのコミットの変更ファイル一覧を展開
- ファイルをクリック: コミット時点との差分を Diff ビューで表示
- デフォルト表示件数: 直近 50 件（「さらに読み込む」で追加）

## 8. Rust バックエンド実装

### 8.1 git2 クレートの使用

Git 操作は Tauri バックエンド（Rust）で `git2` クレートを使用して実装する。外部 `git` コマンドへの依存を避け、クロスプラットフォームで安定動作させる。

```toml
# Cargo.toml
[dependencies]
git2 = "0.18"
```

### 8.2 Tauri コマンド

```rust
// src-tauri/src/git.rs

#[tauri::command]
pub fn git_status(workspace_root: String) -> Result<Vec<FileStatus>, String> {
    let repo = Repository::open(&workspace_root)
        .map_err(|e| e.to_string())?;
    let statuses = repo.statuses(None)
        .map_err(|e| e.to_string())?;

    let result = statuses.iter().map(|entry| FileStatus {
        path: entry.path().unwrap_or("").to_string(),
        status: map_status(entry.status()),
    }).collect();

    Ok(result)
}
```

### 8.3 Git 状態のポーリング

```typescript
// src/store/gitStore.ts
// ファイル保存時・ウィンドウフォーカス時に Git 状態を更新
const GIT_POLL_INTERVAL = 30_000; // 30秒ごとに自動更新

function useGitStatusPoller(workspaceRoot: string | null) {
  useEffect(() => {
    if (!workspaceRoot) return;
    const update = () => invoke('git_status', { workspaceRoot }).then(gitStore.setStatuses);
    update();
    const id = setInterval(update, GIT_POLL_INTERVAL);
    return () => clearInterval(id);
  }, [workspaceRoot]);
}
```

## 10. 制約事項・フォールバック

| 制約 | 対処 |
|------|------|
| ワークスペースが Git リポジトリでない場合 | Git 機能全体を非表示（エラーなし） |
| `git2` 初期化失敗（権限エラー等） | ステータスバーに警告アイコン、機能を無効化 |
| 大規模リポジトリでの `git status` 遅延 | タイムアウト（5 秒）でキャンセル、キャッシュ値を使用 |
| Submodule には非対応 | Submodule ディレクトリは通常ディレクトリとして表示 |
| SSH 認証・HTTPS 認証 | push/pull 非対応のため不要（将来実装時に検討） |
