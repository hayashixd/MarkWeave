# Typora 機能分析レポート

> 分析対象: Typora (v1.x系)
> 目的: 本プロジェクトのマークダウンエディタ開発における参考・指針とする

---

## 1. Typora とは

Typora はシンプルさと機能性を両立した WYSIWYG マークダウンエディタ。

- **開発元**: Typora Team（中国）
- **対応OS**: Windows / macOS / Linux
- **価格**: 有料（$14.99 買い切り、ベータ期間は無料）
- **最大の特徴**: ソースとプレビューを一体化した「シームレスライブエディタ」

### 核心コンセプト

> "A markdown editor, markdown reader"

マークダウン記法を入力した瞬間にレンダリングされ、ソース/プレビューを切り替える必要がない。

## 2. 編集体験（UX）の分析

### 2.1 インラインレンダリング（WYSIWYG）

| 動作 | 詳細 |
|------|------|
| リアルタイム変換 | `**bold**` と入力すると即座に **bold** 表示 |
| フォーカス時ソース表示 | カーソルが当たった箇所だけ生マークダウン表示 |
| フォーカス外レンダリング | カーソルが離れると即座にレンダリング |

この「フォーカスでソース ↔ フォーカス外でレンダリング」の切り替えが Typora の根幹。

### 2.2 ソースモード

- `Ctrl+/` でソースモードと WYSIWYG モードを切り替え可能
- パワーユーザー向けに生マークダウンを直接編集できる
- シンタックスハイライト付き

### 2.3 フォーカスモード / タイプライターモード

| モード | 説明 |
|--------|------|
| フォーカスモード | 現在のパラグラフのみ明るく表示し、他は薄暗く |
| タイプライターモード | カーソル行が常に画面中央に固定される |
| 組み合わせ可能 | 両モードを同時に有効化できる |

## 4. テーブル編集（最重要機能）

Excelライク操作の核心。

### 4.1 テーブルのインタラクション

```
| 操作 | 方法 |
|------|------|
| セル移動 | Tab / Shift+Tab / 矢印キー |
| 行追加（末尾） | 最終行でTab |
| 列幅調整 | ヘッダーのドラッグ |
| 行の追加/削除 | 右クリックコンテキストメニュー |
| 列の追加/削除 | 右クリックコンテキストメニュー |
| 行の並び替え | ドラッグ&ドロップ |
| 列の並び替え | ドラッグ&ドロップ |
| 配置（左/中/右） | ヘッダーのアイコン |
```

### 4.2 テーブル挿入方法

- メニューから行・列数を指定してGUI挿入
- マークダウン記法 `| head |` を入力してEnterで自動生成
- コンテキストメニューから挿入

### 4.3 テーブルのソースとレンダリング

編集中は視覚的なテーブルUIとして表示され、保存時には整形されたマークダウンとして保存される。

## 11. Typora の優れた設計判断（学ぶべき点）

### 11.1 「フォーカス時ソース」モデル

カーソルがある行・要素だけ生マークダウンを表示し、それ以外はレンダリング表示する。これにより：
- 編集したい箇所はソースで操作可能
- 全体の見た目はWYSIWYGで確認可能
- ソース/プレビュー切替の煩わしさが不要

### 11.2 テーブルのファーストクラスサポート

テーブルは他のブロック要素と違い、専用のUIコンポーネントとして実装。行・列操作が豊富でリアルタイム。

### 11.3 コンテキストメニューの充実

右クリックで適切な操作が表示される（テーブル操作、リスト操作等）。

### 11.4 マークダウン入力からの自動変換

- `# ` でH1変換
- `- ` でリスト変換
- `> ` で引用変換
- ` ``` ` でコードブロック変換

これらをリアルタイムで処理することで入力の流れを壊さない。

### 11.5 ファイルをまたいだ操作

ファイルツリーからドラッグ&ドロップでリンク挿入、画像の自動コピー等、ファイルシステムとの統合。

## 12. Typora の制限・課題（改善の余地）

| 課題 | 詳細 |
|------|------|
| 有料 | $14.99の買い切りだが無料ではない |
| プラグインAPI非公開 | 公式プラグインシステムが存在しない |
| コラボレーション非対応 | リアルタイム共同編集なし |
| バージョン管理非内蔵 | Git等は外部ツールに頼る |
| モバイル非対応 | iOS/Android版なし |
| Webアクセス不可 | デスクトップアプリのみ |
| カスタム構文拡張困難 | 独自拡張の追加が限定的 |

## 13. 本プロジェクトへの示唆

### 実装すべきコア機能（MVP）

1. **インラインレンダリング** - フォーカス時ソース、フォーカス外レンダリング
2. **テーブル編集UI** - Tab移動、行/列の追加削除、ドラッグ並び替え
3. **マークダウン自動変換** - `# `, `- `, `> ` 等の入力トリガー
4. **基本的な書式ショートカット** - Ctrl+B/I/K 等
5. **アウトラインパネル** - 見出し一覧からのナビゲーション

### 差別化できる改善点

1. **Web/クラウド対応** - ブラウザで動作、クラウド保存
2. **コラボレーション** - リアルタイム共同編集
3. **プラグインAPI** - 公開されたプラグインシステム
4. **Git統合** - バージョン管理の内蔵
5. **モバイル対応** - レスポンシブデザイン

### 技術的な実装優先度

```
高 ┌─────────────────────────────────────┐
   │ フォーカス/アンフォーカスのWYSIWYG   │
   │ テーブル行・列のCRUD                │
   ├─────────────────────────────────────┤
中 │ コードブロック・数式のレンダリング    │
   │ 画像ドラッグ&ドロップ               │
   │ アウトラインパネル                  │
   ├─────────────────────────────────────┤
低 │ フォーカスモード・タイプライターモード│
   │ テーマシステム                      │
   │ エクスポート（PDF/HTML）            │
低 └─────────────────────────────────────┘
```
