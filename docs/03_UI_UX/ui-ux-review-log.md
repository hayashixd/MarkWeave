## 4. 総評（エディタとしての現在地）

- **基礎編集機能は実用ライン**に到達している（Phase 1〜2 相当）。
- とくに IME ガード、自動保存、テーブル操作など、日常利用で体感差が出る部分は強い。
- 一方で、Typora系エディタとして期待値の高い **「情報探索（アウトライン/検索）」「高度な要素（数式/図/画像）」「入力補助（Slash/Quick Open）」** が未実装で、長文・技術文書の生産性はまだ伸びしろが大きい。

## 5. 標準実装として優先すべき機能（Must Backlog）

### P0（次リリース候補）
1. 数式（KaTeX）インライン・ブロック
2. 画像ドラッグ&ドロップ / クリップボード貼り付け
3. 検索（Ctrl+F）
4. アウトラインパネル（見出しジャンプ）

**理由**: 技術文書エディタとしての最低期待値に直結し、競合比較でも必須機能。

### P1
1. 置換（単一/全置換）
2. Mermaid 図表
3. リンクUI改善（`window.prompt` からダイアログへ）
4. 競合検出時のユーザー選択UI（外部変更時）

### P2
1. エクスポート導線の強化（HTML/PDF プリセット）
2. HTML編集との往復品質改善
3. セッション破損時リカバリ（クラッシュ復旧）

## 6. UI/UX向上のために必要な機能（Should Backlog）

1. **クイックオープン（Ctrl+P）**
   - ファイル名ファジー検索 + 最近開いたファイル
2. **スラッシュコマンド（`/`）**
   - 見出し、表、コード、数式、画像などを高速挿入
3. **コマンドパレット**
   - ショートカット記憶に依存しない操作導線
4. **コードブロック補助UI**
   - 言語選択、コピーボタン、行番号切替
5. **リンク操作UX強化**
   - ホバー時プレビュー、Ctrl+Click 遷移、編集ポップオーバー
6. **フォーカス/Zen系UX**
   - 集中モード、タイプライターモードの段階導入
7. **アクセシビリティ運用強化**
   - キーボードのみ完結の操作監査、ARIA回帰テスト
