## 1. ツールバー UI 設計

### 1.1 全体レイアウト

```
┌──────────────────────────────────────────────────────────────────────────┐
│ [B] [I] [S] [~] [<>]  │  [H1▼] [¶▼] [≡▼]  │  [🔗] [🖼] [⊞]  │  右端  │
│  インライン書式          │  ブロック構造          │  挿入             │        │
└──────────────────────────────────────────────────────────────────────────┘
```

右端エリア（ツールバー右寄せ）:
```
│  [AI コピー ▼]  │  [Typora式 ▼]  │  [🔍]  │
│  AI 機能         │  表示モード       │  検索  │
```

### 1.2 ボタン定義

**インライン書式グループ:**

| ボタン | アイコン | 機能 | ショートカット |
|--------|---------|------|--------------|
| Bold | **B** | 太字 | Ctrl+B |
| Italic | *I* | 斜体 | Ctrl+I |
| Strikethrough | S̶ | 取り消し線 | Alt+Shift+5 |
| Inline Code | `<>` | インラインコード | Ctrl+Shift+` |
| Link | 🔗 | リンク挿入ダイアログ | Ctrl+K |

**ブロック構造グループ:**

| ボタン | 機能 |
|--------|------|
| H1〜H6 ▼ | 見出しレベル選択ドロップダウン |
| ¶ ▼ | 段落・引用・リスト種別選択 |
| ≡ ▼ | テーブル挿入・コードブロック挿入 |

**挿入グループ:**

| ボタン | 機能 |
|--------|------|
| 🖼 | 画像挿入ダイアログ |
| ⊞ | テーブル挿入（行列数グリッド） |
| ∑ | 数式ブロック挿入 |

### 1.3 モード別ツールバー変化

| モード | ツールバー状態 |
|--------|-------------|
| Typora 式 / 常に WYSIWYG | フル表示 |
| 常にソース表示 | インライン書式グループを無効化（グレーアウト） |
| スプリット | フル表示（左ペインの操作に対応） |
| HTML 編集モード | [html-editor-analysis.md](./html-editor-analysis.md) の HTML 専用ツールバーに置換 |

### 1.4 レスポンシブ対応

ウィンドウ幅が狭い場合（< 600px）:
- ボタンラベルを非表示（アイコンのみ）
- 低優先度ボタンをオーバーフローメニュー（`...`）に格納

## 2. メニューバー設計

### 2.1 Tauri ネイティブメニュー構造

Tauri の `Menu` API を使用してOS ネイティブのメニューバーを構築する。

#### ファイル (File)

```
ファイル
├── 新規ファイル              Ctrl+N
├── 開く...                  Ctrl+O
├── フォルダを開く...         Ctrl+Shift+O
├── 最近使ったファイル ▶
│   ├── 1. README.md
│   ├── 2. note.md
│   └── すべてクリア
├── ─────────────────────
├── 保存                     Ctrl+S
├── 名前を付けて保存...       Ctrl+Shift+S
├── ─────────────────────
├── エクスポート ▶
│   ├── HTML にエクスポート...  Ctrl+Shift+E
│   └── PDF にエクスポート...
├── ─────────────────────
├── 印刷...                  Ctrl+P（OS メニュー）
└── アプリを終了             Alt+F4 / Cmd+Q
```

#### 編集 (Edit)

```
編集
├── 元に戻す                 Ctrl+Z
├── やり直す                 Ctrl+Shift+Z
├── ─────────────────────
├── 切り取り                 Ctrl+X
├── コピー                   Ctrl+C
├── 貼り付け                 Ctrl+V
├── プレーンテキストとして貼り付け  Ctrl+Shift+V
├── ─────────────────────
├── すべて選択               Ctrl+A
├── ─────────────────────
├── 検索...                  Ctrl+F
├── 検索と置換...            Ctrl+H
├── フォルダ内検索...        Ctrl+Shift+F
├── ─────────────────────
└── 設定...                  Ctrl+,
```

#### 表示 (View)

```
表示
├── エディタモード ▶
│   ├── Typora 式（デフォルト）  Ctrl+Alt+1
│   ├── 常に WYSIWYG           Ctrl+Alt+2
│   ├── 常にソース表示          Ctrl+Alt+3
│   └── スプリット表示          Ctrl+Alt+4
├── ─────────────────────
├── サイドバーの表示/非表示    Ctrl+Shift+L
├── アウトラインパネル         Ctrl+Shift+1
├── ファイルリスト             Ctrl+Shift+2
├── ファイルツリー             Ctrl+Shift+3
├── ─────────────────────
├── フォーカスモード           F8
├── タイプライターモード       F9
├── フルスクリーン             F11
└── ─────────────────────
    実際のサイズ              Ctrl+0
    拡大                     Ctrl++
    縮小                     Ctrl+-
```

#### ヘルプ (Help)

```
ヘルプ
├── キーボードショートカット一覧
├── ─────────────────────
├── バージョン情報
└── フィードバックを送る...
```

### 2.2 macOS 固有の対応

- **アプリケーションメニュー** (App Name): 「バージョン情報」「設定...」を追加
- `Cmd` キーを `Ctrl` キーの代わりに使用（Tauri の OS 判定で自動切り替え）
- `Cmd+Q` で終了

## 3. エディタ領域コンテキストメニュー設計

### 3.1 テキスト選択時

```
右クリック（テキスト選択中）
├── 切り取り
├── コピー
├── 貼り付け
├── ─────────────────────
├── 太字にする              Ctrl+B
├── 斜体にする              Ctrl+I
├── リンクを挿入...         Ctrl+K
├── ─────────────────────
├── コードとしてマーク
├── 書式をクリア            Ctrl+\
├── ─────────────────────
└── テキスト整形 ▶
    ├── 行を昇順ソート
    ├── 行を降順ソート
    ├── 重複行を削除
    ├── ─────────────────
    ├── 行頭の空白・タブを削除
    ├── 行末の空白・タブを削除
    ├── ─────────────────
    ├── 大文字に変換
    ├── 小文字に変換
    ├── ─────────────────
    ├── 半角→全角変換
    └── 全角→半角変換
```

### 3.2 テキスト非選択時（カーソル位置による）

**段落・見出し上:**
```
右クリック
├── 段落の種類 ▶
│   ├── 見出し H1〜H6
│   ├── 段落（通常テキスト）
│   ├── 引用ブロック
│   └── コードブロック
├── ─────────────────────
├── 上に段落を挿入
├── 下に段落を挿入
└── ─────────────────────
    行を削除
```

**テーブルセル上:**（[system-design.md](./system-design.md) §6 の設計に準拠）
```
右クリック（テーブルセル内）
├── 行を上に挿入
├── 行を下に挿入
├── 列を左に挿入
├── 列を右に挿入
├── ─────────────────────
├── この行を削除
├── この列を削除
├── ─────────────────────
├── セル配置 ▶
│   ├── 左寄せ
│   ├── 中央揃え
│   └── 右寄せ
└── テーブルを削除
```

**画像上:**
```
右クリック（画像上）
├── 画像を別のファイルとして保存...
├── 画像パスをコピー
├── alt テキストを編集...
├── ─────────────────────
└── 画像を削除
```

### 3.3 実装方針

TipTap の `contextmenu` イベントをリッスンし、カーソル位置から対象ノード種別を判定して適切なメニューを表示する。共通コンテキストメニューコンポーネント（`src/components/common/ContextMenu.tsx`）を使用する。

## 4. ステータスバー設計

### 4.1 レイアウト

```
┌──────────────────────────────────────────────────────────────────────┐
│ 左端                                              右端               │
│ 行 12, 列 34  |  選択: 45 文字  |  全体: 1,234 字  約 3 分  |  UTF-8  LF  |  Typora 式 │
└──────────────────────────────────────────────────────────────────────┘
```

### 4.2 表示項目

| 項目 | 内容 | 更新タイミング |
|------|------|--------------|
| 行・列番号 | カーソルの現在位置 | セレクション変化時 |
| 選択文字数 | テキスト選択中のみ表示 | セレクション変化時 |
| 文字数 | ドキュメント全体の文字数（空白・改行含む） | 入力変化時（デバウンス 300ms） |
| 読了時間 | 「約 N 分」（200 wpm 換算） | 同上 |
| エンコーディング | UTF-8 / UTF-8 BOM / Shift-JIS 等 | ファイルオープン時 |
| 改行コード | LF / CRLF | ファイルオープン時 |
| エディタモード | Typora 式 / WYSIWYG / ソース / スプリット | モード切替時 |

### 4.3 クリックアクション

| 項目 | クリック時の動作 |
|------|--------------|
| 行・列番号 | 行番号入力ダイアログ（ジャンプ先行番号を入力） |
| 文字数 | 詳細文書統計ダイアログを開く（§ [text-statistics-design.md](./text-statistics-design.md) 参照） |
| エンコーディング | **エンコーディング操作ポップオーバー**（Reload / Convert の 2 アクション。詳細: [file-workspace-design.md](../04_File_Workspace/file-workspace-design.md) §10.3）|
| 改行コード | **改行コード操作ポップオーバー**（Convert and Save / Change Setting の 2 アクション。詳細: [file-workspace-design.md](../04_File_Workspace/file-workspace-design.md) §11.3）|
| エディタモード | モード切り替えドロップダウン |

## 5. コマンドパレット設計

### 5.1 UI 設計

`Ctrl+Shift+P`（macOS: `Cmd+Shift+P`）で開くコマンド検索パレット。VS Code スタイル。

```
┌───────────────────────────────────────────────────────────┐
│ > コマンドを入力...                                       │
├───────────────────────────────────────────────────────────┤
│  📄 ファイルを開く...                        Ctrl+O       │
│  📋 AI コピー（最適化してコピー）                         │
│  🔍 検索と置換...                           Ctrl+H       │
│▶ ⚙ 設定を開く...                           Ctrl+,        │  ← 選択中
│  🌙 ダークモードに切り替え                              │
│  📊 文書統計を表示                                       │
└───────────────────────────────────────────────────────────┘
           ↑↓ で移動  Enter で実行  Esc で閉じる
```

### 5.3 プレフィックスモード

| プレフィックス | 動作 |
|---|---|
| `>` | コマンド検索（デフォルト） |
| 入力なし | ファイル名ファジー検索（クイックオープンと同等） |
| `@` | 現在ドキュメントの見出し検索（アウトライン）|
| `#` | 現在ワークスペース内のタグ検索（将来機能） |

## 6. フルスクリーンモード設計

### 6.1 動作仕様

- `F11` で Tauri ウィンドウをフルスクリーンに切り替える
- `Escape`（または `F11` 再押下）でフルスクリーン解除

```typescript
// src/app.tsx
import { getCurrentWindow } from '@tauri-apps/api/window';

async function toggleFullscreen() {
  const appWindow = getCurrentWindow();
  const isFullscreen = await appWindow.isFullscreen();
  await appWindow.setFullscreen(!isFullscreen);
}
```

### 6.2 フルスクリーン時の UI 変化

| 要素 | フルスクリーン時の状態 |
|------|------------------|
| タイトルバー（OS ネイティブ） | 非表示（OS により自動） |
| メニューバー | マウスを上端に移動すると一時表示（macOS 互換） |
| ツールバー | 表示維持 |
| ステータスバー | 表示維持 |
| サイドバー | 前の表示状態を維持 |

### 6.3 フォーカスモード（F8）との組み合わせ

フォーカスモード（現在段落以外を薄暗く表示）はフルスクリーンと独立して動作する。
組み合わせた場合: フルスクリーン + 薄暗いオーバーレイの両立。

## 7. サイドバーレイアウト・リサイズ設計

### 7.1 サイドバー構成

```
┌──────────────────────────────────────────────────────────┐
│ [アウトライン] [ファイルリスト] [ファイルツリー]           │  ← パネル切替タブ
├──────────────────────────────────────────────────────────┤
│ （選択中パネルのコンテンツ）                              │
└──────────────────────────────────────────────────────────┘
```

### 7.2 幅リサイズ

サイドバーと本文エディタの境界に **ドラッグハンドル** を配置する。

```typescript
// src/components/Sidebar/ResizableSidebar.tsx
const [sidebarWidth, setSidebarWidth] = useState(240); // デフォルト 240px
const minWidth = 160;
const maxWidth = 500;

// pointerdown → pointermove → pointerup でリサイズ
```

- 最小幅: 160px / 最大幅: 500px
- リサイズ後の幅を `settingsStore` に保存（`sidebarWidth: number`）

### 7.3 サイドバーの表示/非表示

- `Ctrl+Shift+L` でトグル
- 非表示時は幅 0 でアニメーション退場（`transition: width 200ms ease`）
- 表示状態は `settingsStore` に保存（`sidebarVisible: boolean`）

### 7.4 パネル切り替え

| ショートカット | パネル |
|---|---|
| `Ctrl+Shift+1` | アウトライン |
| `Ctrl+Shift+2` | ファイルリスト |
| `Ctrl+Shift+3` | ファイルツリー |

## 8. 初回起動・オンボーディング設計

### 8.1 初回起動判定

```typescript
// src/onboarding/onboarding.ts
const isFirstLaunch = !(await store.get('onboarding.completed'));
```

### 8.2 ウェルカム画面

初回起動時のみ、エディタの代わりにウェルカム画面を表示する。

```
┌────────────────────────────────────────────────────────┐
│                                                        │
│         🖊  Markdown Editor へようこそ                  │
│                                                        │
│   ┌────────────────┐    ┌────────────────┐             │
│   │ 📄 新規作成    │    │ 📂 ファイルを開く│            │
│   └────────────────┘    └────────────────┘             │
│                                                        │
│   ┌────────────────────────────────────┐               │
│   │ 📁 フォルダを開いてワークスペースを作成│            │
│   └────────────────────────────────────┘               │
│                                                        │
│   最近使ったファイル:                                   │
│   • README.md  - 2026-02-24                            │
│   • note.md    - 2026-02-23                            │
│                                                        │
│   [ チュートリアルを開く ]     [ スキップ ]             │
│                                                        │
└────────────────────────────────────────────────────────┘
```

### 8.3 チュートリアル

チュートリアルは **インタラクティブなサンプルドキュメント** として実装する。
実際に編集しながら機能を体験できる形式にする。

```markdown
# Markdown Editor チュートリアル

このドキュメントを編集しながら機能を体験できます。

## 1. 見出しを作る

行頭に `# ` を入力すると見出しになります。
↓ この行を試してみましょう:
見出しにする

## 2. テキスト装飾

- **太字**: Ctrl+B または `**テキスト**`
- *斜体*: Ctrl+I または `*テキスト*`
...
```

### 8.4 チュートリアル完了後

- サンプルドキュメントを閉じ、通常のエディタ画面に戻る
- `store.set('onboarding.completed', true)` を記録
- 2 回目以降の起動ではウェルカム画面をスキップ
- ヘルプメニューから「チュートリアルを再表示」で再実行可能
