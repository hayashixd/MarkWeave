## 1. プロダクト概要

- 製品名: **MarkWeave**
- コンセプト: Write. Polish. Publish. — Webで文章を書いて公開する人のための、ローカルファーストWYSIWYGエディタ
- ターゲット: Markdown で技術記事・ブログ記事を書いて、Zenn / Qiita / dev.to / Hashnode に公開するエンジニア・テクニカルライター
- 提供価値:
  - Typora ライクな書き心地で Markdown をリアルタイム編集
  - プラットフォーム向け公開ワークフロー（Zenn / Qiita 最適化エクスポート）
  - ローカルファースト設計（クラウド不要・ベンダーロックインなし）
  - AI 活用支援（BYOK: ユーザー自身の Claude API キーで動作）

公開用コピー案:

> 書く。整える。公開する。
> Typora ライクな直感操作で Markdown を書いて、技術ブログプラットフォームへワンクリック公開。
> ファイルはすべてローカルに。クラウド不要。

## 2. 主要機能（公開用）

### 2.1 WYSIWYG編集

- Typora 式インラインレンダリング（フォーカス時にソース表示、非フォーカス時にレンダリング）
- 見出し / リスト / 引用 / コード / テーブル / 数式（KaTeX）/ Mermaid 図表をサポート
- 表示モード: WYSIWYG / ソースモード / スプリットモード（サイドバイサイド）
- YAML Front Matter エディタ（記事メタデータの GUI 編集）
- タブで複数ファイルを同時編集

### 2.2 テーブル編集（Excelライク）

- セル移動（Tab / Shift+Tab）
- 行・列の追加 / 削除
- 行・列のドラッグ並び替え
- 列幅リサイズ・配置変更

### 2.3 ファイル・ワークスペース

- ファイルを開く / 保存 / 名前を付けて保存
- フォルダをワークスペースとして開く
- ファイルツリー表示、作成 / 削除 / リネーム
- 外部変更検知（Dropbox・OneDrive 等との安全な共存）

### 2.4 執筆支援モード

- **Zen モード**: メニュー・サイドバーを非表示にして執筆に集中
- **フォーカスモード**: カーソル周辺のみハイライト
- **タイプライターモード**: カーソル行を常に画面中央に配置
- **ポモドーロタイマー**: 作業時間の区切り管理
- **ワードスプリント**: 時間制限付き執筆チャレンジ

### 2.5 文書統計・テーマ

- 文字数 / 読了時間 / 可読性スコアのリアルタイム表示
- テーマ切り替え（ライト / ダーク / その他）

### 2.6 エクスポート

- HTML（スタンドアロン、画像 Base64 埋め込み）
- PDF
- Word / EPUB（Pandoc 連携）
- Markdown（プラットフォーム向け最適化）

### 2.7 AI連携（BYOK）

- **AIコピー**: ドキュメントをAI入力向けに自動最適化してクリップボードへコピー
- **AIテンプレート**: ブログ構成・コード解説・コードレビュー依頼・要約などのテンプレート提供
- ※ 自分の Claude API キーを設定して利用（クラウドサービス不使用）

### 2.8 マニュアル（ユーザーガイド）

- 公式マニュアルをサイト上で参照可能（`doc-public/manuals/user-manual.md` を公開原稿として利用）
- 機能ごとの操作手順とスクリーンショットを掲載
- アプリ更新に合わせて内容を継続更新

## 4. 導入情報（公開用）

### 必要環境（開発・ビルド時）

- Node.js 20+
- pnpm 9+
- Rust 1.77.2+
- Tauri 前提ライブラリ（OS依存）

### 開発向け起動コマンド（必要最小限）

- `pnpm install`
- `pnpm dev`（フロントのみ）
- `pnpm tauri dev`（デスクトップアプリ）

### 注意事項（ユーザー向け）

- Windows 版は現在 Authenticode 署名なし。インストール時に SmartScreen 警告が表示される場合があります（「詳細情報 → 実行」で回避可能）。
- macOS は現在非対応です。

## 8. 追加掲載を検討すべき内容（提案）

現行の公開情報は機能紹介が中心のため、公式サイトとしては以下の情報を追加すると
「誰向けの製品か」「なぜ選ぶべきか」「導入判断に必要な情報」が伝わりやすくなる。

### 8.1 想定ユーザー / ユースケース

- Markdown で技術記事を書いて Zenn / Qiita / dev.to / Hashnode に公開するエンジニア
- WYSIWYG で書きたいが Markdown の資産は失いたくないテクニカルライター
- コードブロック・図表・数式を多用する執筆者

公開用コピー案:

> Markdown は書けるけど、もっと快適に書きたい。
> MarkWeave は「書き心地」と「公開ワークフロー」にフォーカスした、エンジニアのためのエディタです。

### 8.2 差別化ポイント（競合比較をしない表現）

- **ローカルファースト**: ファイルはすべてローカルに保存。クラウド同期不要。iCloud / Dropbox / OneDrive に委ねる。
- **買い切りモデル**: サブスクなし。一度購入すれば追加費用なし（予定: $24.99、3デバイスまで）。
- **AI BYOK**: 自分の Claude API キーを設定して AI 機能を利用。API コストの押しつけなし。
- **プラットフォーム公開ワークフロー**: Zenn / Qiita 向け Markdown 最適化エクスポートを標準搭載。
- **大規模ファイル対応**: サイズに応じた表示モード最適化（3MB 超でソースモード自動切替）。

公開用コピー案:

> 買い切り・ローカルファースト・AI BYOK。
> 月額費用なし。クラウド不要。AIも自分のキーで。

### 8.3 セキュリティ / データ取り扱い方針（公開可能な範囲）

- 文書はローカルファイルを中心に管理
- 外部サービス依存を前提としない設計
- AI 機能は BYOK（ユーザーの API キーで動作。Anthropic との通信はユーザーの責任範囲）
- 内部運用詳細（鍵管理、CI 設定など）は非公開
