Scenario 01 — Zenn / Qiita

Zenn / Qiita に
週1本記事を書いてクロスポスト

Before

VS Code のプレビューとエディタを行き来するたびに書くリズムが崩れる。Zenn で書いた :::message や @[youtube] を Qiita 向けに手作業で書き直すのが面倒。

After

プロファイルを「Zenn」に設定して書くだけ。Qiita 非対応記法があればリアルタイム警告。「⇄ Qiita 用に変換してコピー」で変換済み Markdown がクリップボードに入り、そのまま Qiita に貼り付け完了。

  1. Ctrl+N で新規ファイルを作成し、FM パネルでプロファイルを Zenn に切り替え
  2. title / emoji / type / topics を GUI 入力。YAML を直接書かなくてよい
  3. Zenn 記法パレットから :::message や @[youtube] をワンクリック挿入しながら執筆
  4. FM パネルの「📋 Markdown をコピー」で Zenn 向け完全 Markdown をクリップボードへ → Zenn に貼り付けて公開
  5. Qiita にもポストしたい場合は「⇄ Qiita 用に変換してコピー」をクリック。Zenn 固有記法を自動除去した Markdown が取得できる
WYSIWYG 編集 プラットフォームプロファイル Zenn 記法パレット 構文警告 Qiita 変換コピー

Scenario 02 — AI Copy

コードスニペット入り記事を
AI にレビューしてもらう

Before

Claude / ChatGPT に記事を貼るたびに手作業で整形。見出しのレベルがズレたり、コードブロックの言語タグが抜けていたり。

After

「AI コピー」ボタン1クリックで見出し補正・言語タグ付け・余分な空行削除を自動処理。クリップボードをそのまま貼り付けるだけ。

  1. コードブロックに言語名を指定して記事を書く（シンタックスハイライトがリアルタイムで確認できる）
  2. 記事を書き終えたらツールバーの「AI コピー」ボタンをクリック
  3. 整形済みのMarkdownがクリップボードにコピーされる
  4. Claude / ChatGPT に貼り付けてレビューや改善を依頼
  5. フィードバックを受けてエディタに戻り修正 → 再度エクスポート
AI コピー コードブロック（40+ 言語） シンタックスハイライト HTML エクスポート

Scenario 03 — Focus Writing

集中して
長文を書く

Before

ツールバーやサイドバーが気になって集中できない。30分書いてどこまで進んだかわからなくなる。

After

Zen モードでUIを完全非表示、タイプライターモードで視点を画面中央に固定。ポモドーロで25分ごとに区切り、文字数・読了時間をリアルタイムで把握。

  1. 「表示 → Zen モード」（F11）でサイドバー・ツールバーを非表示に
  2. 「表示 → タイプライターモード」でカーソル行を画面中央に固定
  3. 「ツール → ポモドーロ」で25分タイマーをスタート
  4. 集中して書く。ステータスバーの文字数・読了時間が進捗の目安に
  5. 25分経ったら休憩。ワードスプリントで目標文字数チャレンジも可能
Zen モード フォーカスモード タイプライターモード ポモドーロタイマー 文書統計

Scenario 04 — Workspace

既存の .md ファイルを
引き継いで管理する

Before

クラウドエディタに取り込むとフォーマットが崩れる。既存の .md ファイルをそのまま使えるエディタがなかった。Dropbox で管理しているファイルが勝手に上書きされることがある。

After

既存の .md ファイルをそのまま開いてWYSIWYGで編集。フォルダをワークスペースとして開けば記事一覧をサイドバーで管理。外部変更は自動上書きせずに通知。

  1. 「ファイル → フォルダを開く」で Zenn・Qiita 記事フォルダをワークスペースとして開く
  2. サイドバーのファイルツリーから編集したい記事をクリック
  3. 既存の Markdown がそのまま WYSIWYG で表示される
  4. Ctrl+P（クイックオープン）でフォルダ内を検索して素早く開く
  5. Dropbox / OneDrive で同期中のファイルが外部変更された場合は通知が届き、上書きするかどうかを選択できる
ワークスペース ファイルツリー クイックオープン 外部変更検知 ローカルファースト
