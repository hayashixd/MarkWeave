# 高度な機能ガイド

基本的な使い方は [user-manual.md](./user-manual.md) を参照してください。
ここでは上級者向けの機能・すべてのメニュー項目・完全なショートカット一覧を説明します。

---

## 1. ワークスペースとファイル管理

### フォルダをワークスペースとして開く

メニュー「ファイル → フォルダを開く...」（`Ctrl+Shift+O`）

フォルダをワークスペースとして開くと:

- 左サイドバーにファイルツリーが表示されます
- フォルダ内のファイル作成・削除・リネームができます
- クイックオープン（`Ctrl+P`）でフォルダ内のファイルを素早く開けます
- ファイルをリネームすると、ドキュメント内のリンクが自動で更新されます

**外部変更の検知**: Dropbox・OneDrive 等でファイルが外部から変更された場合、自動上書きはしません。「エディタの内容を保持」か「ディスクから再読み込み」かを選択できます。

### 最近使ったファイル・最近のワークスペース

- メニュー「ファイル → 最近使ったファイル...」: 最近開いたファイルをリストから素早く開きます
- メニュー「ファイル → 最近のワークスペース...」: 最近使ったワークスペース（フォルダ）に切り替えます

### デイリーノート

メニュー「ファイル → デイリーノート作成」（`Ctrl+Alt+D`）

今日の日付名（例: `2026-03-16.md`）でファイルを作成します。ワークスペース使用時は、ワークスペースの `daily/` フォルダに作成されます。毎日のメモや作業ログを継続的に記録するのに便利です。

### テンプレートから新規作成

メニュー「ファイル → テンプレートから新規作成...」

定義済みのファイルテンプレートを元に新規ファイルを作成します。AI テンプレートパネルとは独立した、ファイル構造のテンプレート機能です。

### 別名で保存（形式変換）

| 操作 | メニュー | ショートカット |
|------|---------|-------------|
| Markdown として保存 | ファイル → 別名で保存 → Markdown として保存... | `Ctrl+Shift+M` |
| HTML として保存 | ファイル → 別名で保存 → HTML として保存... | `Ctrl+Shift+H` |

「エクスポート」はオプション付きのダイアログで出力するのに対し、「別名で保存」は現在の内容をシンプルに別形式で保存します。

### 印刷

メニュー「ファイル → 印刷...」（`Ctrl+P`）

OS の標準印刷ダイアログを使用して印刷します。

## 2. サイドバーパネル

### サイドバーの表示/非表示

メニュー「表示 → サイドバーの表示/非表示」（`Ctrl+Shift+L`）

サイドバー全体をトグルで非表示にして、編集領域を広げることができます。

### 各パネルのショートカット

| パネル | メニュー | ショートカット |
|--------|---------|-------------|
| アウトライン | 表示 → アウトラインパネル | `Ctrl+Shift+1` |
| ファイルツリー | 表示 → ファイルパネル | `Ctrl+Shift+2` |
| AI テンプレート | 表示 → AI テンプレート | `Ctrl+Shift+3` |
| バックリンク | 表示 → バックリンク | `Ctrl+Shift+4` |
| タグビュー | 表示 → タグビュー | `Ctrl+Shift+5` |
| Git パネル | 表示 → Git パネル | `Ctrl+Shift+7` |

### バックリンク

メニュー「表示 → バックリンク」（`Ctrl+Shift+4`）

現在のファイルを参照している他のファイルの一覧を表示します。Wikiリンク記法（`[[ファイル名]]`）で相互参照しているドキュメントを管理するのに便利です。

> **有効化方法**: 設定（`Ctrl+,`）→「プラグイン」タブ → バックリンクを有効化

### タグビュー

メニュー「表示 → タグビュー」（`Ctrl+Shift+5`）

YAML Front Matter や本文中の `#タグ` を収集して一覧表示します。タグをクリックすると、そのタグを持つファイルを絞り込み表示できます。

> **有効化方法**: 設定（`Ctrl+,`）→「プラグイン」タブ → タグビューを有効化

### Git パネル

メニュー「表示 → Git パネル」（`Ctrl+Shift+7`）

ワークスペース内の Git リポジトリの変更状態（変更ファイル・ステージング・差分）を表示します。Git がインストールされている必要があります。

> **有効化方法**: 設定（`Ctrl+,`）→「プラグイン」タブ → Git パネルを有効化

## 3. フローティング目次

メニュー「表示 → フローティング目次」（`Ctrl+Shift+T`）

エディタ内に小さなフローティングウィンドウで目次を表示します。アウトラインパネルとは異なり、サイドバーを閉じた状態や Zen モード中でも目次を確認・ナビゲートできます。

---

## 4. 分割エディタ

メニュー「表示 → ペイン分割」（`Ctrl+\`）

エディタを左右に分割して、2つのファイルを同時に表示・編集できます。

- 分割線をドラッグしてペインのサイズを調整
- タブをドラッグして別ペインに移動
- `Ctrl+Alt+←` / `Ctrl+Alt+→` でペイン間のフォーカスを切り替え
- 同一ファイルを両ペインで開くとスクロール位置が同期されます

## 5. 執筆支援モード

### フォーカスモード

メニュー「表示 → フォーカスモード」（`F8`）

現在編集中の段落のみをハイライトし、前後の文章をグレーアウトします。長文執筆で集中力を維持するのに有効です。

### タイプライターモード

メニュー「表示 → タイプライターモード」（`F9`）

カーソル行を常に画面中央に配置します。視線移動が少なくなり、長時間の執筆に適しています。

### Zen モード

メニュー「表示 → Zen モード」（`F11`）

サイドバー・ツールバー・ステータスバーをすべて非表示にして執筆に集中するモードです。Zen モード中は環境音（ホワイトノイズ・雨音・カフェの音など）を再生することもできます（設定 → 執筆タブ）。

## 7. 執筆ツール

### ポモドーロタイマー

画面下部のステータスバーにあるポモドーロアイコンをクリックして起動します。

25分作業 + 5分休憩のサイクルで集中時間を管理します。タイマーをクリックして開始・一時停止・リセットができます。

### ワードスプリント

画面下部のステータスバーにあるワードスプリントアイコンをクリックして起動します。

時間制限内（例: 10分）で目標文字数を達成するチャレンジ機能です。進捗がリアルタイムで表示されます。

### 文書統計

メニュー「編集 → 文書統計...」

文字数・単語数・段落数・推定読了時間・可読性スコアなどを確認できます。ステータスバーには文字数と読了時間がリアルタイムで常時表示されています。

## 8. AI 機能

### AI コピー

ツールバーの「✨ AI コピー」ボタンを押すと、現在のドキュメントを AI 入力向けに最適化した形式でクリップボードにコピーします。Claude / ChatGPT などにそのまま貼り付けてレビューや改善を依頼する際に便利です。

最適化の内容（個別にオン/オフ可能）:

- 見出し階層の修正
- コードブロックへの言語タグ付与
- リスト記号の統一
- 過剰な空白行の削除
- リンクへの URL 注記追加

### AI テンプレート

メニュー「表示 → AI テンプレート」（`Ctrl+Shift+3`）

用途別のプロンプトテンプレートを選択・エディタに挿入できます。

| カテゴリ | 内容例 |
|---------|--------|
| ブログ | 記事構成・導入文・まとめ |
| コード | コードレビュー依頼・コード解説 |
| 要約 | 文章の箇条書き化・要約 |
| 翻訳 | 日英翻訳プロンプト |
| 議事録 | 会議メモテンプレート |

カスタムテンプレートをパネル右上の「+」ボタンから作成することもできます。本文中の `{{変数名}}` がプレースホルダーになります。

## 10. YAML Front Matter

ファイル先頭の `---` ブロックを GUI で編集できます。エディタ上部の FM バーをクリックして展開します。

```yaml
---
title: "TypeScript で型安全な API クライアントを作る"
emoji: "🛡️"
type: "tech"
topics: ["typescript", "api"]
published: false
---
```

Zenn・Qiita のタイトル・タグ・公開設定の管理に便利です。

## 13. 大規模ファイルの扱い

MarkWeave は大容量ファイルにも対応しています:

- **50KB 超**: UI ペイント後に非同期で遅延処理（入力レイテンシを妨げない）
- **3MB 超**: ソースモードに自動切替（WYSIWYG パースを回避して高速化）

通常の技術記事（数十 KB）では意識する必要はありません。

## 14. FAQ（上級）

**Q. バックリンクやタグビューはどこで有効にしますか？**
設定（`Ctrl+,`）→「プラグイン」タブから有効化できます。デフォルトは非表示です。

**Q. Git パネルを使うには？**
Git がインストールされている環境でフォルダをワークスペースとして開き、設定 → プラグインタブで有効化してください。

**Q. ポモドーロやワードスプリントのショートカットはありますか？**
現在はショートカットキーはありません。画面下部のステータスバーのアイコンから起動できます。

**Q. クラウド同期はできますか？**
MarkWeave 自体はクラウド同期機能を持ちません。iCloud・Dropbox・OneDrive などの OS 標準のファイル同期と組み合わせてご利用ください。

**Q. 大きいファイルは扱えますか？**
はい。3MB 超のファイルはソースモードに自動切替して対応します。

**Q. Word / EPUB へのエクスポートに失敗します。**
Word (.docx)・LaTeX・EPUB エクスポートには Pandoc のインストールが必要です。[pandoc.org](https://pandoc.org/installing.html) からインストールしてください。
